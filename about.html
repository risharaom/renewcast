What's really running behind these pages.

The real thing, running for real

Unlike an earlier version of this site, the Forecast, Clustering and Country pages here call an actual Python backend (/backend) that runs your notebook's real pipeline: the same pandas cleaning and lag-feature engineering, the same RandomForestRegressor(n_estimators=100, random_state=42) for both models, and the same KMeans(n_clusters=4) clustering.

If that backend isn't running, every page falls back to a lightweight JS approximation instead (so the site never breaks) — look for the "Live Python model" vs. "Preview data" badge at the top of each page to see which one you're looking at.

Starting the real model

Run run_backend.sh (Mac/Linux) or run_backend.bat (Windows) once — it sets up Python automatically and starts the API at http://localhost:8000. Then open index.html as usual. See README.md for the full walkthrough.

About the bundled dataset

This project ships with a ready-to-use starter dataset (backend/data/energy.csv) shaped exactly like the real Kaggle "world-energy-transition-2000-2025" dataset, so it runs immediately with zero setup. Swap in the real Kaggle file any time — see README.md for how.
